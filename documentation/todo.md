## Task list

### Using this doc

- **Goal**: a single task list that can hold anything (bugs, refactors, “rework X”, research, chores).
- **Rule of thumb**:
  - Put brand-new ideas in **Inbox**
  - Keep **Next** small (5–10 items)
  - Only a few items belong in **In progress**
  - Move finished items to **Done** with a date

### Task template (copy/paste)

- [ ] (P2) Short title
  - **Type**: bug | feature | rework | refactor | perf | balance | ux | test | build | docs | research
  - **Area**: save/load | levelgen | combat | UI | rendering | input | audio | etc
  - **Why**: one sentence
  - **Notes**: optional bullets
  - **Acceptance**:
    - [ ] concrete check 1
    - [ ] concrete check 2
  - **Links**: file paths / issues / screenshots / repro steps
